# Задание 1

## Задача 1

### Текст задачи

Разработать класс с тремя целочисленными полями. Создать конструктор копирования.
Разработать метод, вычисляющий произведение полей. Перегрузить метод ToString()
для формирования строки из полей класса. Реализовать дочерний класс.
Протестировать все конструкторы и другие методы базового и дочернего классов.

### Алгоритм решения

Базовый класс `Character` три целочисленных поля:
- `damage` — базовый урон персонажа (больше 0)
- `critMultiplier` — множитель критического удара
- `attackSpeed` — скорость атаки (больше 0)

Реализованно:
- конструктор с параметрами, использующий свойства для проверки значений
- конструктор копирования, копирующий поля напрямую
- свойства `Damage`, `CritMultiplier`, `AttackSpeed` — для `Damage` и `AttackSpeed`
реализована проверка: значение должно быть больше 0, иначе выбрасывается
`ArgumentOutOfRangeException`
- метод `GetTotalDamage()` — возвращает произведение всех трёх полей
- перегруженный метод `ToString()` — возвращает строку с значениями полей

Дочерний класс `Mage` наследует `Character` и добавляет два поля:
- `mana` — запас маны (не меньше 0)
- `spellDamage` — урон заклинания (не больше 100)

`Mage` реализует:
- конструктор с параметрами, вызывающий конструктор базового класса через `base`
- конструктор копирования, вызывающий конструктор копирования базового класса через `base`
- свойства `Mana` и `SpellDamage` с проверками значений
- метод `CastSpell()` — применяет заклинание, тратит 10 единиц маны и возвращает
урон заклинания; если маны недостаточно — выводит сообщение и возвращает 0
- метод `IsOutOfMana()` — возвращает `true` если маны меньше 10
- перегруженный метод `ToString()` — дополняет строку базового класса полями мага

Для ввода данных используется вспомогательный класс `HelpConsole` из пространства
имён `Checkup`, содержащий метод `ReadInt()`, который проверяет корректность
введённого значения на соответствие типу `int`.

# Задание 2

## Задача 1

### Текст задачи

Реализовать класс `Time` с полями `byte hours` и `byte minutes`. Разработать
конструкторы, свойства, перегрузить метод `ToString()`. Реализовать метод
вычитания переменной типа `Time` из объекта типа `Time` с учётом перехода
в предыдущие сутки. Протестировать все методы.

### Алгоритм решения

Класс `Time` содержит два поля типа `byte`:
- `hours` — часы (0–23)
- `minutes` — минуты (0–59)

Класс реализует:
- конструктор с параметрами, использующий свойства для проверки значений
- конструктор копирования, копирующий поля напрямую
- свойства `Hours` и `Minutes` с проверками: при выходе за допустимый диапазон
выбрасывается `ArgumentOutOfRangeException`
- перегруженный метод `ToString()` — возвращает время в формате `HH:MM`
- метод `Subtract(Time obj)` — вычитает переданный объект из текущего; оба времени
переводятся в минуты, вычисляется разность, применяется формула
`((result % 1440) + 1440) % 1440` для корректного перехода в предыдущие сутки,
результат возвращается как новый объект `Time`

Для ввода данных в `HelpConsole` добавлен метод `ReadByte()`, проверяющий
корректность введённого значения на соответствие типу `byte`.

# Задание 3

## Задача 1

### Текст задачи

Добавить к классу `Time` перегруженные операции: унарные `++` и `--`,
операции приведения типа к `int` (неявная) и `bool` (явная),
бинарные операции `<` и `>`. Протестировать все операции.

### Алгоритм решения

К классу `Time` добавлены следующие перегруженные операции:

Унарные операции:
- `++` — увеличивает время на 1 минуту; реализовано через перевод в общее
количество минут, прибавление 1 и обратное преобразование с учётом перехода суток
- `--` — аналогично, но вычитает 1 минуту

Операции приведения типа:
- `implicit operator int` — неявное приведение; возвращает общее количество минут
по формуле `hours * 60 + minutes`
- `explicit operator bool` — явное приведение; возвращает `true` если хотя бы одно
из полей не равно нулю, иначе `false`

Бинарные операции:
- `operator <` — сравнивает два объекта `Time` через неявное приведение к `int`;
возвращает `true` если левый операнд меньше правого
- `operator >` — аналогично, возвращает `true` если левый операнд больше правого
